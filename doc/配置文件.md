在多模块 Maven 项目中，MyBatis 配置文件（如 `mybatis-config.xml`、Mapper XML 文件、`mybatis-config.yml` 等）的存放位置需遵循**“职责隔离 + 工程规范”** 原则，结合你给出的模块分工，以下是最优配置方案：

### 一、核心结论（按配置类型分）
| 配置文件类型                | 推荐存放模块       | 具体路径                          | 核心原因                                                                 |
|-----------------------------|--------------------|-----------------------------------|--------------------------------------------------------------------------|
| `mybatis-config.xml`（全局配置） | `common` 模块      | `common/src/main/resources/mybatis/mybatis-config.xml` | 全局配置（如别名、插件、环境）是所有模块共用的，`common` 作为公共模块最合理 |
| Mapper XML 文件（如 `UserMapper.xml`） | `dal` 模块         | `dal/src/main/resources/mapper/*.xml` | `dal`（数据访问层）负责数据库交互，Mapper XML 是数据层核心，归属于 dal  |
| `mybatis-plus.yml`/`application.yml`（Spring Boot 整合配置） | `web` 或 `service` 模块 | `web/src/main/resources/application.yml` | 运行时配置（如数据源、Mapper 扫描路径）随启动模块（通常 web 是启动模块） |

### 二、分模块详细配置说明
#### 1. `common` 模块：存放 MyBatis 全局通用配置
`common` 是全项目公共依赖模块，适合存放所有模块共用的 MyBatis 全局配置（非运行时环境配置）：
```
common/
└── src/main/resources/
    └── mybatis/
        └── mybatis-config.xml  # 全局配置：别名、插件、类型处理器等
```
**示例 `mybatis-config.xml` 内容**：
```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE configuration
        PUBLIC "-//mybatis.org//DTD Config 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-config.dtd">
<configuration>
    <!-- 全局别名（所有模块共用） -->
    <typeAliases>
        <package name="com.xxx.common.entity"/>
    </typeAliases>
    <!-- 通用插件（如分页插件、驼峰转换） -->
    <plugins>
        <plugin interceptor="com.github.pagehelper.PageInterceptor">
            <property name="helperDialect" value="mysql"/>
        </plugin>
    </plugins>
</configuration>
```

#### 2. `dal` 模块：存放 Mapper XML + 数据层专属配置
`dal`（Data Access Layer）是数据访问层，负责数据库交互，**所有和 SQL 相关的 Mapper XML 必须放在这里**，同时可存放 dal 模块专属的 MyBatis 配置：
```
dal/
└── src/main/resources/
    ├── mapper/  # Mapper XML 目录（按业务分目录更佳）
    │   ├── user/
    │   │   └── UserMapper.xml
    │   └── order/
    │       └── OrderMapper.xml
    └── mybatis/
        └── dal-mybatis-config.xml  # 可选：dal 模块专属配置（如特定类型处理器）
```
**关键：Mapper 接口与 XML 对应**  
`dal` 模块的 Mapper 接口放在 `dal/src/main/java/com/xxx/dal/mapper/`，XML 放在 `dal/src/main/resources/mapper/`，保证路径匹配（如 `UserMapper.java` 对应 `mapper/user/UserMapper.xml`）。

#### 3. `web` 模块：存放 Spring Boot 整合配置（运行时）
若项目是 Spring Boot 架构，`web` 通常是**启动模块**（包含 `Application` 启动类），因此 MyBatis 与 Spring 的整合配置（如数据源、Mapper 扫描路径）放在 `web` 模块的 `application.yml`/`application.properties` 中：
```
web/
└── src/main/resources/
    └── application.yml  # Spring Boot 配置文件
```
**示例 `application.yml` 中 MyBatis 配置**：
```yaml
mybatis:
  # 引用 common 模块的全局配置
  config-location: classpath:mybatis/mybatis-config.xml
  # 扫描 dal 模块的 Mapper XML
  mapper-locations: classpath:mapper/**/*.xml
  # 扫描 dal 模块的 Mapper 接口
  type-aliases-package: com.xxx.dal.entity
spring:
  datasource:
    # 数据源配置（运行时核心，随启动模块放 web）
    url: jdbc:mysql://localhost:3306/test?useUnicode=true&characterEncoding=utf8
    username: root
    password: 123456
    driver-class-name: com.mysql.cj.jdbc.Driver
```

### 三、模块依赖关系（保证配置可被加载）
需确保模块间依赖正确，否则配置文件无法被读取：
```xml
<!-- web 模块依赖 service，service 依赖 dal，dal 依赖 common -->
<!-- web/pom.xml -->
<dependencies>
    <dependency>
        <groupId>com.xxx</groupId>
        <artifactId>service</artifactId>
    </dependency>
</dependencies>

<!-- service/pom.xml -->
<dependencies>
    <dependency>
        <groupId>com.xxx</groupId>
        <artifactId>dal</artifactId>
    </dependency>
</dependencies>

<!-- dal/pom.xml -->
<dependencies>
    <dependency>
        <groupId>com.xxx</groupId>
        <artifactId>common</artifactId>
    </dependency>
    <!-- MyBatis 核心依赖放在 dal 模块（数据层专属） -->
    <dependency>
        <groupId>tk.mybatis</groupId>
        <artifactId>mapper</artifactId>
    </dependency>
    <dependency>
        <groupId>org.mybatis</groupId>
        <artifactId>mybatis</artifactId>
    </dependency>
</dependencies>
```

### 四、特殊场景补充
1. **无 `dal` 模块（数据层逻辑在 `service`）**：  
   若项目未拆分 `dal`，Mapper XML 可放在 `service/src/main/resources/mapper/`，全局配置仍放 `common`。
2. **多环境配置（dev/test/prod）**：  
   不同环境的数据源配置可放在 `web/src/main/resources/config/` 下（如 `application-dev.yml`），通过 `spring.profiles.active` 激活。
3. **纯 MyBatis（非 Spring Boot）**：  
   需在 `dal` 模块中配置 `SqlSessionFactory`，指定 `mybatis-config.xml` 和 Mapper XML 路径，示例：
   ```java
   SqlSessionFactory sessionFactory = new SqlSessionFactoryBuilder()
       .build(Resources.getResourceAsReader("mybatis/mybatis-config.xml"));
   ```

### 五、核心原则总结
1. **“谁用谁存”**：数据层配置（Mapper XML）归 `dal`，全局通用配置归 `common`，运行时配置归启动模块（`web`）；
2. **“依赖可传递”**：确保配置文件所在模块被启动模块依赖，否则无法加载；
3. **“路径规范”**：资源文件统一放在 `src/main/resources/`（非 `static` 目录！`static` 是 Web 静态资源（如 JS/CSS），MyBatis 配置不能放这里）。

**关键避坑**：不要把 MyBatis 配置放在 `static` 目录（`static` 是 Spring Boot 暴露给前端的静态资源目录，MyBatis 无法扫描该目录下的配置文件），必须放在 `src/main/resources/` 根目录或子目录下。
